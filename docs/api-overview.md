# 🔌 API & UI Documentation

## Роль API в системі
Наш API — це внутрішній міст (**Internal API**) між базою даних та клієнтами:
1. **Telegram Bot**: Смикає `/validate` для пропуску людей.
2. **Web Dashboard**: Смикає `/analytics` для малювання графіків розробникам RichYou.

## UI Документація (Storybook)
Для фронтенду ми використовуємо **Storybook**. Це бібліотека компонентів, де кожен елемент можна потестити окремо.

### Основні компоненти:
- **ScannerResponse**: Картка, що вискакує в боті (Зелена — "Ок", Червона — "Бан").
- **StatCard**: Віджет для дашборду з цифрами відвідуваності.
- **LoginField**: Поле вводу з валідацією JWT-токена.
