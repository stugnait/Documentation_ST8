# 🤖 CI/CD та Автоматизація

Ми впровадили автоматичний конвеєр документації, щоб не витрачати час на ручне оновлення.

## Pipeline кроки:
1. **Linting**: Кожен `push` перевіряє `openapi.yaml` на помилки (Swagger-cli).
2. **Build Docs**: 
   - Генерація статичного сайту через `mkdocs`.
   - Збірка **Swagger UI** для інтерактивного тестування API.
3. **Deploy**: Автопублікація на **GitHub Pages** після мержу в `main`.

## Тригери:
- **Push у гілку `feature/*`**: Тільки перевірка валідності.
- **Merge у `main`**: Повний деплой оновленої документації.
